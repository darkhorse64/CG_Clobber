Présentation du jeu

Ce jeu est une implémentation du jeu de plateau Clobber.

Le code source est accessible ici.

Je tiens à remercier struct qui m'a donné l'autorisation de réutiliser son code pour ce projet.

  Règles

Le jeu se déroule sur un échiquier de 8x8 cases. Dans la position initiale, toutes les cases sont occupées par un pion, les pions blancs sur les cases blanches, les pions noirs sur les cases noires.

Comment jouer:
  • Lors de son tour de jeu, un joueur sélectionne un de ses pions et "démolit" un pion de son adversaire situé sur une case adjacente (horizontalement ou verticalement). Le pion qui a été "démoli" est retiré du plateau et est remplacé par le pion sélectionné.

La partie se termine quand un joueur, lors de son tour de jeu, n'a plus de coups disponibles (aucun de ses pions n'est voisin d'un pion adverse). Le joueur a alors perdu la partie.

Fin d'un tour de jeu:
  • Le robot doit communiquer les coordonnées du pion qu'il a choisi suivi des coordonnées de sa destination. Les coordonnées sont exprimées en notation échiquéenne, par exemple e2e3.
En cas de coup erroné d'un joueur:
  • Si les positions de départ ou d'arrivée sont en dehors du plateau,
  • Si la case de départ est occupée par un pion adverse,
  • Si la case d'arrivée est vide ou occupée par un des pions sur,

la partie se termine et le joueur perd.

  Règles pour les experts

Les parties nulles sont impossibles.
Conditions de victoire
  • Etre le dernier joueur à pouvoir déplacer un pion.
Conditions de défaite
  • Ne plus avoir de coups à sa disposition.
  • Ne pas répondre dans le temps imparti ou envoyer une sortie erronée.

  Entrées/sorties du jeu

Entrées initiales
Première ligne: boardSize: le nombre de lignes et de colonnes du plateau.
Ligne suivante: color: la couleur de vos pions (w: blancs, b:noirs).
Entrées pour chaque tour de jeu
boardSize lignes: line: une chaîne de caractères représentant une ligne du plateau, de haut en bas. (.: vide, w: pion blanc, b: pion noir).
Ligne suivante: lastAction: le dernier coup joué par l'adversaire ("null" si c'est le premier tour).
Ligne suivante: actionsCount: le nombre de coups légaux pour ce tour.
Sorties
Une seule ligne contenant les coordonnées de départ et d'arrivée du pion joué, par exemple "e2e3".
  • Vous pouvez afficher un (court) message dans le cartouche de votre joueur en ajoutant un espace après le coup suivi du texte de votre message, par exemple "e2e3 message".
  • Vous pouvez jouer un coup aléatoirement en écrivant random à la place de votre coup. Ca n'a vraiment d'utilité que pour créer facilement un robot fonctionnel à l'aide du générateur de code.

Contraintes
Le temps de réponse au premier tour est ≤ 1000 ms.
Le temps de réponse pour les tours suivants est ≤ 150 ms.
